sidekiq-unique-jobs is a Sidekiq middleware gem that uses Redis locks to ensure only one copy of a job with the same arguments runs at a time. Whether you need to prevent duplicate enqueuing, duplicate execution, or both, this gem provides the tools to enforce job uniqueness throughout the entire lifecycle.

## How it works

sidekiq-unique-jobs intercepts jobs at enqueue time and execution time using Sidekiq middleware. When a job is pushed to Redis, the gem computes a digest from the job's class, queue, and arguments, then attempts to acquire a Redis lock for that digest. If the lock is already held, the configured conflict strategy takes effect — by default, the duplicate is silently dropped.

<Steps>
  <Step title="Install the gem">
    Add `sidekiq-unique-jobs` to your Gemfile and run `bundle install`.
  </Step>
  <Step title="Configure middleware">
    Register the client and server middleware in your Sidekiq initializer.
  </Step>
  <Step title="Add lock type to your job">
    Set `sidekiq_options lock: :until_executed` on any Sidekiq worker class.
  </Step>
  <Step title="Observe and tune">
    Use the Web UI, reflection hooks, and configuration options to monitor and fine-tune lock behavior.
  </Step>
</Steps>
